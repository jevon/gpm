How It Works

GPM enhances your development workflow by creating rich context for your dependencies.

1

Package Installation

GPM detects your project type and installs packages using the appropriate manager (npm, pip, or gem/bundler).

2

Package Research

GPM researches the package by fetching data from registries and GitHub, extracting examples and patterns.

3

Context Creation

A structured context is created in .gpm/<package>/research.json, optimized for your detected AI coding assistant.

4

MCP Server Generation

GPM generates an Express server in .gpm/<package>/mcp/ with API endpoints that provide package context.

5

LLM Integration

AI coding assistants can access the MCP server endpoints to understand the package better, resulting in more accurate code generation.
